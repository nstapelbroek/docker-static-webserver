Using environment variables

When this container bootstraps, a script will find and replace all occurrences of ${VARIABLE_NAME} and replace them for their matching values passed in the environment variables of your container. This is great when trying to ship a webpack compiled static page in a Docker image on multiple environments.

As an example: a container is almost certain to receive a hostname when running it. The hostname of this container is ${HOSTNAME}. Try running another instance of this container and notice how the hostname changes.
